# 3.1 Queue Wait Analysis

## Summary
Added comprehensive queue wait analysis to the Performance tab on the workflow detail page. All computations are client-side from existing `runs` data — no new API endpoints, no additional GitHub API calls, no rate limit impact.

## Features
- **Queue stat cards:** Avg wait, p95 wait (with SLA breach indicator at >5min), delayed run count, developer time wasted estimate
- **Day×Hour heatmap:** 7×24 grid (UTC) showing average queue wait by day-of-week and hour-of-day, color-coded from green (low) to red (high), with hover tooltips
- **Queue distribution histogram:** Buckets (<10s, 10s–30s, 30s–1m, 1m–2m, 2m–5m, 5m–10m, >10m)
- **Queue wait trend chart:** Area chart showing wait time per run over time
- **Branch impact table:** Top 10 branches ranked by queue impact — avg wait, p95, delayed count, estimated wasted developer-minutes
- **Developer cost estimate:** Total wait hours and estimated USD cost at $75/hr

## Files Created
- `src/lib/queue-analysis.ts` — Pure computation utilities: `computeQueueStats`, `computeQueueHeatmap`, `computeBranchQueueImpact`, `computeQueueDistribution`, `computeQueueTrend`, `estimateQueueCost`

## Files Modified
- `src/app/repos/[owner]/[repo]/workflows/[workflow_id]/page.tsx`
  - Added import for queue-analysis utilities
  - Extended `PerformanceTab` props to include `runs: WorkflowRun[]`
  - Added queue analysis section after slowest steps table (stat cards, heatmap, distribution, trend, branch table)
  - Added `QueueHeatmap` component (day×hour grid with colour normalisation)

## Architecture Decisions
- **Client-side only:** Same pattern as DORA metrics — reuses existing `runs` data already fetched by the page
- **No new API endpoints:** Zero additional network calls
- **Conditionally rendered:** Queue analysis section only shows when `queueStats.total > 0`
- **UTC timestamps:** Heatmap uses UTC to match GitHub API timestamps
- **SLA threshold:** 5 minutes default, configurable in utility functions
