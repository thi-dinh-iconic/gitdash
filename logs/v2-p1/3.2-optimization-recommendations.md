# 3.2 Workflow Optimization Recommendations

## Summary
Added a rule-based optimization engine that analyzes workflow run data and surfaces actionable tips on the Overview tab. Tips are dismissible per-session. All analysis is client-side — no new API endpoints, no additional GitHub API calls.

## Features
- **10 optimization rules** covering cost, performance, reliability, and security:
  1. High average queue wait (>2 min)
  2. Queue wait SLA breach (p95 >5 min)
  3. Low success rate (<80%)
  4. High re-run rate (>10%, flakiness indicator)
  5. Duration increasing trend (>20% increase recent vs older)
  6. High cancellation rate (>15%)
  7. Long average duration (>10 min)
  8. Weekend run concentration (>20%, cost optimization)
  9. Active failure streak (3+ consecutive failures)
  10. Timed-out runs (2+ timeouts)
- **Severity levels:** Critical (red), Warning (amber), Info (blue) — sorted by severity
- **Category badges:** Cost, Performance, Reliability, Security
- **Impact estimates:** Each tip includes estimated impact where quantifiable
- **Dismissible:** Each tip has an X button to dismiss it for the session
- **Tip count badge:** Shows number of active suggestions

## Architecture Decisions
- **Client-side only:** Same pattern as DORA and Queue Analysis — pure computation on existing runs data
- **No persistence:** Dismissed tips reset on page reload (no localStorage needed; tips are re-evaluated each visit)
- **Stable tip IDs:** Each rule has a fixed ID string for dismissal tracking
- **Conservative thresholds:** Rules only fire when there's enough data (minimum 5-20 runs depending on rule)
