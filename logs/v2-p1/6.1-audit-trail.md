# 6.1 Audit Trail

## Summary

Workflow file change history page for each repository. Shows commits that modified files under `.github/workflows/`, with author info, timestamps, and links to the commit on GitHub. Includes stat cards (total changes, contributors, last 24h changes, files tracked) and a recent-changes alert banner.

No database required — uses the GitHub Commits API filtered to the `.github/workflows/` path.

## Files Created

### `src/lib/github.ts` (additions)
- `WorkflowFileCommit` type — sha, message, date, author info, file_path, html_url
- `listWorkflowFileCommits(octokit, owner, repo, limit)` — fetches commits touching `.github/workflows/` via the Commits API, then extracts per-file entries from each commit

### `src/app/api/github/audit-log/route.ts`
- API route: `GET /api/github/audit-log?owner=X&repo=X&limit=50`
- Validates owner/repo via `validateOwner`/`validateRepo`
- Returns `WorkflowFileCommit[]` sorted newest-first
- Cache-Control: `private, s-maxage=120, stale-while-revalidate=300`

### `src/app/repos/[owner]/[repo]/audit/page.tsx`
- Full audit trail page with:
  - Breadcrumb navigation via `RepoWorkflowBreadcrumb`
  - Refresh button + "Workflows on GitHub" external link
  - Recent changes alert (amber banner for changes in last 24h)
  - 4 stat cards: Total Changes, Contributors, Last 24h, Files Tracked
  - Commit timeline with avatars, messages, author, time ago, file path, commit SHA link
- Uses SWR for data fetching
- `getRecentChanges()` helper defined outside component to avoid React 19 purity lint rule
