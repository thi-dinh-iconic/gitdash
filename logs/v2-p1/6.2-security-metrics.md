## What was built

### `src/app/api/github/security-scan/route.ts`
- Fetches `.github/workflows/` directory listing via `repos.getContent`
- Downloads and decodes (base64) each YAML file
- Runs regex-based static analysis — no external dependencies
- Returns `SecurityScanResponse` with per-file findings and aggregate scores
- Cache-Control: `private, s-maxage=300, stale-while-revalidate=300`

#### Security Rules Implemented
| Rule ID | Severity | Check |
|---------|----------|-------|
| SEC-001 | Critical | `pull_request_target` trigger used |
| SEC-002 | Critical | Secret echoed to logs (`echo ${{ secrets.X }}`) |
| SEC-003 | High | Secret passed as CLI argument (`--flag=${{ secrets.X }}`) |
| SEC-004 | Critical | Secret written to `$GITHUB_OUTPUT` |
| SEC-005 | High | Unquoted `workflow_dispatch` input in `run:` |
| SEC-006 | High | Action pinned to mutable ref (`@main`, `@master`, `@HEAD`) |
| SEC-007 | Medium | No `timeout-minutes` set |
| SEC-008 | Medium | No `permissions:` block declared |
| SEC-009 | Info | `workflow_dispatch` inputs without `type:` declaration |
| SEC-010 | Medium | `amazon-ecr-login` without `mask-password: true` |

#### Scoring
- Start at 100, deduct: critical −30, high −15, medium −7, info −2
- Per-file score + overall average across all scanned files
