## Features Delivered

### A. Overall DORA Performance Level
- Computed as the worst of the four individual metrics (industry standard)
- Color-coded banner: Elite (green), High (blue), Medium (amber), Low (red)
- Visual level indicator bars

### B. Four Metric Cards
Each card shows:
- Metric name + icon
- Current value (human-readable)
- Performance level badge (Elite/High/Medium/Low)
- Supporting detail (sample size, count, etc.)
- Industry benchmark for the current level

**1. Deployment Frequency**
- Calculated as: completed runs / period days
- Labels: "X/day", "X/week", "X/month"
- Benchmarks: Elite (multiple/day), High (daily-weekly), Medium (weekly-monthly), Low (<monthly)

**2. Lead Time for Changes**
- Approximated as: queue wait + execution time (created_at → completion)
- Shows median and p95
- Benchmarks: Elite (<1hr), High (<1day), Medium (<1week), Low (>1month)

**3. Change Failure Rate**
- Calculated as: failure runs / total completed runs × 100
- Shows raw numbers (X failures / Y total)
- Benchmarks: Elite (0-5%), High (0-15%), Medium (16-30%), Low (>30%)

**4. Mean Time to Recovery (MTTR)**
- Calculated per-branch: time from first failure → next success
- Shows average across all recovery events
- Benchmarks: Elite (<1hr), High (<1day), Medium (<1week), Low (>1week)

### C. Benchmark Reference Table
- Full comparison table showing all four metrics × all four levels
- Highlights current performance level for each metric
- Based on DORA State of DevOps Report

## Architecture Decision: Client-Side Computation

The DORA metrics are computed entirely client-side from the existing `runs` data already fetched by the workflow detail page. This approach was chosen because:

1. **No additional API calls** — the runs data is already available
2. **No new API endpoint needed** — reduces surface area
3. **Instant tab switching** — no loading state for DORA tab
4. **Consistent with existing pattern** — Reliability and Triggers tabs also compute from runs
5. **No rate limit impact** — zero additional GitHub API calls
